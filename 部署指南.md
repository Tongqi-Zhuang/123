# Netlify 部署指南 - 木鱼应用

本指南将详细说明如何将木鱼应用部署到 Netlify，以便在手机上使用。

## 📋 准备工作

### 方法一：通过 GitHub 部署（推荐，最简单）

#### 第一步：创建 GitHub 账号（如果还没有）
1. 访问 https://github.com
2. 点击右上角 "Sign up" 注册账号
3. 完成邮箱验证

#### 第二步：安装 Git（如果还没有安装）
1. 访问 https://git-scm.com/download/win
2. 下载并安装 Git for Windows
3. 安装时保持默认选项即可
4. 安装完成后，重启 Cursor 编辑器

#### 第三步：初始化 Git 仓库并上传到 GitHub

**在 Cursor 中操作：**

1. **打开终端**（在 Cursor 中按 `Ctrl + ~` 或点击菜单 View → Terminal）

2. **初始化 Git 仓库**（在终端中输入以下命令，每行按回车）：
   ```bash
   git init
   git add .
   git commit -m "Initial commit: 木鱼应用"
   ```

3. **在 GitHub 创建新仓库**：
   - 登录 GitHub
   - 点击右上角 "+" 号 → "New repository"
   - Repository name 输入：`wooden-fish-fufu`（或任意名称）
   - 选择 Public（公开）或 Private（私有）
   - **不要**勾选 "Initialize this repository with a README"
   - 点击 "Create repository"

4. **连接本地仓库到 GitHub**（在终端中执行，将 YOUR_USERNAME 替换为你的 GitHub 用户名）：
   ```bash
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/wooden-fish-fufu.git
   git push -u origin main
   ```
   - 如果提示输入用户名和密码，用户名是你的 GitHub 用户名
   - 密码需要使用 Personal Access Token（见下方说明）

**获取 GitHub Personal Access Token：**
- 登录 GitHub
- 点击右上角头像 → Settings
- 左侧菜单选择 Developer settings
- 选择 Personal access tokens → Tokens (classic)
- 点击 Generate new token (classic)
- Note 填写：`Netlify Deploy`
- 勾选 `repo` 权限
- 点击 Generate token
- **复制生成的 token**（只显示一次，请保存好）
- 在 Git 推送时，密码处粘贴这个 token

#### 第四步：在 Netlify 部署

1. **访问 Netlify**：
   - 打开 https://www.netlify.com
   - 点击右上角 "Sign up" 注册（可以用 GitHub 账号直接登录）

2. **导入项目**：
   - 登录后，点击 "Add new site" → "Import an existing project"
   - 选择 "Deploy with GitHub"
   - 授权 Netlify 访问你的 GitHub 账号
   - 选择你刚才创建的仓库 `wooden-fish-fufu`

3. **配置部署设置**（通常会自动识别，但请确认）：
   - Build command: `npm run build`
   - Publish directory: `dist`
   - 点击 "Deploy site"

4. **等待部署完成**：
   - Netlify 会自动开始构建和部署
   - 通常需要 1-3 分钟
   - 部署完成后，你会看到一个绿色的 "Published" 提示
   - 网站会有一个自动生成的网址，类似：`https://random-name-123.netlify.app`

5. **自定义域名（可选）**：
   - 点击 "Site settings" → "Change site name"
   - 可以修改为更友好的名称，如：`wooden-fish-fufu`
   - 最终网址会是：`https://wooden-fish-fufu.netlify.app`

---

### 方法二：直接拖拽部署（更简单，但无法自动更新）

如果你不想使用 Git，可以直接拖拽部署：

1. **构建项目**：
   - 在 Cursor 终端中运行：`npm run build`
   - 等待构建完成（会生成 `dist` 文件夹）

2. **访问 Netlify**：
   - 打开 https://www.netlify.com
   - 注册/登录账号

3. **拖拽部署**：
   - 登录后，直接将项目中的 `dist` 文件夹拖拽到 Netlify 的部署区域
   - 等待上传和部署完成

**注意**：使用此方法后，每次修改代码都需要重新构建并拖拽上传。

## 📱 在手机上使用

部署完成后：

1. **在手机浏览器中打开**：
   - 打开手机浏览器（Safari、Chrome、Edge 等）
   - 输入 Netlify 提供的网址
   - 例如：`https://wooden-fish-fufu.netlify.app`

2. **添加到主屏幕**（推荐）：
   - **iPhone (Safari)**：
     - 打开网站后，点击底部分享按钮
     - 选择 "添加到主屏幕"
     - 可以自定义名称和图标
   - **Android (Chrome)**：
     - 打开网站后，点击右上角菜单（三个点）
     - 选择 "添加到主屏幕" 或 "安装应用"
     - 确认添加

3. **使用体验**：
   - 添加后，可以像普通 App 一样从主屏幕打开
   - 支持触摸敲击、震动反馈、音效等功能

## ❓ 常见问题

### 1. 构建失败
- 检查 `package.json` 中的依赖是否正确
- 确保在项目根目录运行 `npm run build`
- 查看 Netlify 的构建日志找出错误原因

### 2. 网站无法访问
- 检查 Netlify 部署状态是否为 "Published"
- 确认网址是否正确
- 尝试清除浏览器缓存

### 3. 手机端显示异常
- 检查 `index.html` 中的 viewport 设置
- 确保使用了响应式设计
- 在手机浏览器中打开开发者工具检查错误

### 4. 音效不工作
- 某些浏览器需要用户交互后才能播放音频
- 确保点击了页面后才测试音效
- 检查浏览器是否允许自动播放音频
